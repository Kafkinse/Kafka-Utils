# Пошагово для новичка: домен + Cloudflare Tunnel

Самый простой способ спрятать релей за Cloudflare, **без проброса портов и без
сертификатов**. Друзья будут подключаться по `wss://chat.твойдомен.xyz`, а IP
твоего VPS не будет виден.

Почему этот способ, а не Caddy из `CLOUDFLARE.md`: тут VPS сам «звонит» в
Cloudflare изнутри, поэтому проброс портов и фаервол вообще не нужны. Для
человека за NAT — идеально.

---

## Что понадобится

- Домен, добавленный в Cloudflare (см. Шаг 0).
- Релей уже работает как сервис (у тебя это `kafka-relay`, порт 8765 — готово).

---

## Шаг 0. Домен и Cloudflare (делается один раз)

1. Купи любой дешёвый домен — например на **porkbun.com** или **namecheap.com**
   (домен `.xyz` часто стоит ~1$ в год). Имя любое, например `mychat.xyz`.
2. Заведи бесплатный аккаунт на **cloudflare.com** → **Add a site** → введи свой
   домен → выбери **Free** план.
3. Cloudflare покажет два адреса **nameserver** (вида `xxx.ns.cloudflare.com`).
   Зайди в панель, где купил домен, и в настройках DNS/nameservers **замени
   текущие на эти два**. Сохрани.
4. Подожди, пока в Cloudflare домен станет **Active** (обычно 10–60 минут,
   иногда дольше). Придёт письмо. Дальше — Шаг 1.

## Шаг 1. Создать туннель в панели Cloudflare

1. Открой **one.dash.cloudflare.com** (это Cloudflare Zero Trust). Первый раз
   попросит выбрать имя команды и план — выбери **Free**, карту вводить не надо.
2. Слева: **Networks → Tunnels** → **Create a tunnel**.
3. Тип коннектора — **Cloudflared** → **Next**.
4. Имя туннеля — любое, например `kafka` → **Save tunnel**.

## Шаг 2. Установить коннектор на VPS (одна команда)

После создания Cloudflare покажет команды установки под разные системы.
Выбери вкладку **Debian** (64-bit) — там будет готовая команда с твоим токеном,
примерно такого вида:

```bash
curl -L --output cloudflared.deb https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb \
  && sudo dpkg -i cloudflared.deb \
  && sudo cloudflared service install ДЛИННЫЙ_ТОКЕН_ИЗ_ПАНЕЛИ
```

**Скопируй команду прямо из панели** (там уже вставлен твой токен) и выполни на
VPS. Она поставит cloudflared и запустит его как службу.

Проверка: в панели Cloudflare внизу этой же страницы коннектор станет
**Connected** (зелёным). Если да — жми **Next**.

## Шаг 3. Привязать адрес к релею

На следующем экране (**Public Hostname**) заполни:

| Поле | Значение |
|---|---|
| Subdomain | `chat` |
| Domain | выбери свой домен из списка (`mychat.xyz`) |
| Path | оставь пустым |
| Type | **HTTP** |
| URL | `localhost:8765` |

Нажми **Save tunnel / Save hostname**.

Cloudflare сам создаст нужную DNS-запись — руками в DNS ничего добавлять не
надо. Полный адрес получится `chat.mychat.xyz`.

## Шаг 4. Настроить мод (у тебя и у всех друзей)

В игре: **X → Friends → Messenger**:

| Настройка | Значение |
|---|---|
| Server URL | `wss://chat.mychat.xyz`  (свой домен, без порта!) |
| Key | тот же секретный ключ, что на релее |

Схема `wss://` и без порта — важно (это TLS через Cloudflare на 443).

## Шаг 5. Проверить

1. Зайди в игре в `/kafka pm`.
2. В шапке окна должно стать зелёное **● релей**, а в чат придёт
   `[Мессенджер] подключено к релею`.
3. Если у друга раньше не подключалось из-за блокировки — теперь заработает,
   потому что соединение идёт на адреса Cloudflare и выглядит как обычный сайт.

---

## Если что-то не так

- **Коннектор не «Connected»** — проверь, что команду из Шага 2 выполнил
  полностью и что VPS в интернете (`ping 1.1.1.1`).
- **В моде `● оффлайн`** — проверь адрес ровно `wss://chat.твойдомен` (без
  `ws://`, без `:8765`) и что релей жив: `sudo systemctl status kafka-relay`.
- **Домен не Active в Cloudflare** — значит nameservers ещё не сменились
  (Шаг 0.3), подожди ещё.
- **Проверка снаружи**: `curl -I https://chat.твойдомен.xyz` должен что-то
  ответить, а не висеть.

## Плюсы этого способа

- Проброс портов на NAT больше не нужен вообще.
- Сменился IP VPS — ничего не меняешь, туннель сам переподключится.
- Реальный IP сервера скрыт, трафик как обычный HTTPS → блокировки по IP/порту
  не мешают.
- Прямой вход `ws://163.123.180.81:10041` можно оставить как запасной.
